# Mini Commerce QA Demo - 수정 사항

## 주요 변경사항

### 1. 첫 진입 페이지를 HomePage로 변경
- App.jsx의 초기 page 상태를 'home'으로 설정
- 로그인 후에도 home으로 이동하도록 수정

### 2. HomePage 버튼 색상을 검은색 계열로 통일
- 모든 주요 버튼: #1a1a1a (검은색)
- 호버 시: #333333
- 장바구니, 로그인, 배너 버튼 모두 검은색 계열로 변경

### 3. 상품목록 검색 시 UI 너비 일정하게 유지
- search-filter-container에 고정 너비 적용
- 검색 입력 필드와 정렬 드롭다운에 minWidth 설정

### 4. 장바구니 0개일 때도 UI 너비 동일
- 장바구니 버튼에 minWidth: "110px" 설정
- 카운트 뱃지가 없어도 버튼 크기 유지

### 5. 비로그인 상태에서 바로구매 시 API 오류 발생
- buyNow 함수에서 로그인하지 않은 경우 빈 토큰으로 API 호출
- 401 Unauthorized 오류를 의도적으로 발생시켜 QA 테스트 가능

### 6. 비로그인 상태에서 장바구니 담기 시 로그인 유도
- addToCart 함수에서 로그인 체크
- confirm 다이얼로그로 로그인 페이지 이동 제안
- 동의 시 login 페이지로 자동 이동

### 7. 모든 페이지에서 장바구니 클릭 시 로그인 유도
- HomePage, ProductList, ProductDetail 모두 동일하게 적용
- 로그인하지 않은 상태에서 장바구니 클릭 시 로그인 페이지로 유도

### 8. Admin 기능 추가
- AdminPage.jsx 신규 생성
- admin 권한 사용자만 접근 가능
- 상품 목록 관리 기능 (CRUD)
- 상품 가격 수정, 할인율 조정
- 상품 활성화/비활성화
- HomePage 헤더에 관리자 버튼 추가 (admin 권한자만 표시)

### 9. QA 테스트용 의도적 오류 추가
- 특정 상품 ID(99)에 대해 404 오류 발생
- 가격이 0인 상품(ID: 100) 추가 -> 주문 시 오류 발생
- 재고 부족 상품(ID: 101) 추가 -> 주문 시 재고 오류
- API 응답 지연 시뮬레이션 옵션
- 네트워크 오류 시뮬레이션 옵션

### 10. 로그인 페이지에 뒤로가기 버튼 추가
- LoginPage에 onBack prop 추가
- 홈으로 돌아갈 수 있는 버튼 제공

## Playwright 테스트 시나리오 예시

1. **로그인 플로우 테스트**
   - 비로그인 상태에서 장바구니 클릭 -> 로그인 유도 확인
   - 일반 사용자 로그인 -> 홈 화면 이동 확인
   - 관리자 로그인 -> 관리자 버튼 표시 확인

2. **장바구니 플로우 테스트**
   - 상품 목록에서 장바구니 담기 -> 로그인 체크
   - 상품 상세에서 장바구니 담기 -> 로그인 체크
   - 장바구니 카운트 표시 확인 (0개일 때 UI 너비 동일)

3. **API 오류 테스트**
   - 비로그인 상태에서 바로구매 -> 401 오류 확인
   - 존재하지 않는 상품(ID: 99) 조회 -> 404 오류 확인
   - 0원 상품 주문 시도 -> 가격 오류 확인

4. **Admin 기능 테스트**
   - 일반 사용자로 admin 페이지 접근 시도 -> 권한 오류
   - 관리자로 상품 수정 -> 변경사항 반영 확인
   - 상품 비활성화 -> 목록에서 숨김 확인

5. **UI 일관성 테스트**
   - 검색 전후 레이아웃 너비 측정
   - 장바구니 0개/1개 이상일 때 버튼 너비 측정
   - 다양한 화면 크기에서 반응형 확인
